## Phase 2 — First feature set

Good first frontend features:

- [ ] Modal/dialog with focus trap and keyboard close.
- [ ] Command palette.
- [ ] Toast notification system.
- [ ] Tabs with keyboard navigation.
- [ ] Virtual list.
- [ ] Resizable split panel.
- [ ] Drag-and-drop sortable list.
- [ ] Infinite scroll with loading/error states.
- [ ] Theme switcher using CSS variables.
- [ ] Form validation patterns.

- [ ] Add accessibility notes per feature.
- [ ] Add performance notes per feature.

## Phase 4 — Open-source operations

- [ ] Add `CONTRIBUTING.md`.
- [ ] Add issue templates.
- [ ] Add PR template.
- [ ] Add CI checks.
- [ ] Add release/changelog workflow.
- [ ] Publish docs/demo site.

## North-star vision

Make `vibeSrc` a friendly frontend source library where developers can learn how polished product UI features are built, understand the trade-offs, and copy reliable patterns into their own apps.
